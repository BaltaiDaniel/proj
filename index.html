Design and Construction of a Face‑Tracking Multi‑Parameter Health
Monitoring System with IoT Integration

1. Introduction

Advancements in computer vision and machine learning have opened new avenues for non‑contact health monitoring. Traditional health assessment relies on contact‑based sensors such as chest straps, finger pulse oximeters and thermometers, which, while accurate, can cause discomfort and restrict continuous monitoring. In critical scenarios such as neonatal care, burn units or infectious disease outbreaks, physical contact may be undesirable or impractical. Contactless techniques based on remote photoplethysmography (rPPG) and facial analysis have emerged as promising alternatives. This study aims to design and construct a face‑tracking health monitor using an ordinary webcam connected to a Raspberry Pi, with the capability to estimate heart rate, breathing rate, emotional state, stress, fatigue and drowsiness. The system will process facial video streams in real time using Python together with OpenCV, NumPy and MediaPipe, and will incorporate a Wi‑Fi module for IoT connectivity, enabling remote data logging and alerts. By combining multiple health indicators through a single non‑invasive sensor, the proposed system offers a low‑cost, scalable solution for remote health monitoring.

2. Problem Statement

The timely detection of abnormal physiological states plays a key role in preventing medical emergencies such as cardiac events, respiratory distress and fatigue‑related accidents. However, many existing health monitors require deliberate user action (e.g., placing a finger on a sensor or wearing a chest strap) and provide only isolated measurements. The discontinuous approach may miss transient anomalies, while continuous contact‑based devices can be uncomfortable for long‑term use. Moreover, many low‑cost monitoring solutions lack the ability to assess multiple health parameters simultaneously, or to transmit data to a healthcare provider. There is therefore a need for a cost‑effective, non‑contact system that can continuously track several vital signs from a single video source and relay this information over the internet for remote observation and alerting.

3. Aim and Objectives

Aim: To design and construct an automated, face‑tracking health monitor that uses a standard webcam and a Raspberry Pi to measure heart rate, breathing rate, emotional state, stress, fatigue and drowsiness in real time, and to transmit these data to an IoT platform.

Objectives:

  1. To extract a region of interest (forehead/cheek) and process the colour variations to compute heart rate and breathing rate via remote photoplethysmography (rPPG).
  2. To analyze facial expressions, eye closure and head pose for emotion recognition, stress, fatigue and drowsiness detection.
  3. To integrate a Wi‑Fi module and upload all health metrics securely to an IoT cloud platform (e.g., Blynk or ThingSpeak).

4. Methodology (Block Diagram)

The system architecture is shown in Figure 1. A webcam continuously captures facial video, which is sent to the Raspberry Pi. Python scripts using OpenCV and MediaPipe detect the face and extract 468 landmarks. For rPPG, the forehead or cheek region is selected and the average green channel intensity is recorded over time. The raw signal is filtered, and a Fast Fourier Transform (FFT) yields heart rate and breathing rate. Simultaneously, MediaPipe provides eye‑aspect ratio (EAR) for drowsiness detection, head pose angles for attention and a facial expression classifier for emotion and stress. All derived metrics (heart rate, breathing rate, emotion, stress/fatigue level, drowsiness/attention) are displayed on a local dashboard. The Wi‑Fi module sends these data to an IoT cloud service (e.g., Blynk or ThingSpeak) for remote storage and visualisation. When a metric exceeds predefined thresholds (e.g., HR < 50 or > 120 bpm; eye closure > 3 s), the system activates a local buzzer/LED and pushes a notification to a caregiver's smartphone.

All modules operate concurrently, ensuring real‑time performance (15–30 FPS).

5. Significance of the Project

This project offers several significant benefits. First, it eliminates physical contact, making health monitoring feasible for users who cannot tolerate wearable sensors (e.g., burn victims, premature infants, elderly with sensitive skin). Second, it integrates multiple health indicators (cardiorespiratory, emotional, attentional) from a single, low‑cost webcam, reducing hardware expenses and complexity. Third, real‑time rPPG processing enables early detection of abnormal heart rates or breathing patterns, while the drowsiness detection module can help prevent fatigue‑related accidents in drivers or machine operators. Fourth, the IoT connectivity allows remote caregivers or family members to monitor a person's wellbeing without being physically present – an invaluable feature for telemedicine and home‑based elderly care. Finally, the open‑source software stack (Python, OpenCV, NumPy, MediaPipe) ensures reproducibility and easy adaptation for further research or clinical applications.

6. Scope of the Project

The scope of this project is limited to the development and testing of a prototype non‑contact health monitor using a Raspberry Pi (any model with sufficient processing power, e.g., Pi 4 or Pi 5) and a standard USB webcam. The system will be designed for single‑person use in a static or semi‑static environment (e.g., seated at a desk or in front of a monitor). Ethical and privacy concerns are acknowledged, and the prototype is intended for research validation only – it is not a certified medical device. The IoT component will be implemented using a free‑tier cloud service (Blynk or ThingSpeak) with Wi‑Fi connectivity; security measures such as API key authentication are considered within best‑effort scope. The system will be tested in typical indoor lighting conditions with skin‑tone variability considered during performance evaluation. No external sensors (e.g., contact oximeter) are required for operation, but such a reference will be used only for validation.

7. Expected Outcomes

Upon completion, the following specific outcomes are anticipated:

  • A working prototype that displays real‑time heart rate (BPM) and breathing rate (breaths/min) on a local screen, alongside emotional state (happy/neutral/sad) and fatigue level (low/medium/high).
  • Visual indicators (coloured LEDs or on‑screen badges) for stress and drowsiness; an audible buzzer alarm triggers when eye closure exceeds 3 seconds (drowsiness) or when HR drifts outside 50–120 BPM.
  • All five primary health metrics (HR, BR, emotion, stress/fatigue, drowsiness/attention) uploaded to an IoT cloud dashboard every 5 seconds, including time‑stamped history.
  • A performance report showing rPPG‑derived heart rate agreement with a finger pulse oximeter within ±5 BPM under good lighting; drowsiness detection accuracy above 90% in simulated low‑alertness trials.
  • Remote notifications delivered to caregiver smartphone (via Blynk or Telegram bot) within 5 seconds of abnormal event detection.
